# Phase 3-B: 完整四季循环系统实现提示词

## 任务概述

在 Prompt 10 的配置系统基础上，将 Demo 0.2 的“春季时间系统”扩展为 Demo 0.3.0 的“全年四季循环系统”。  
本阶段只做**四季循环、季节倍率、轻量视觉与 UI**，不引入粒子系统和完整新作物玩法链。

---

## 核心目标

### 1. TimeManager 扩展为全年系统

在现有基础上新增：

- `season_changed(season_id: String, year_count: int)` 信号
- `year_count: int`
- `current_season_config: SeasonConfig`

并把季节推进逻辑改为：

- `1 刻 = 30 秒`
- `4 刻 = 1 时辰`
- `12 时辰 = 1 天`
- `7 天 = 1 节气`
- `6 节气 = 1 季节`
- `4 季 = 1 年`

季节顺序必须来自：

```gdscript
ConfigManager.get_time_config().seasons_order
```

### 2. 四季倍率驱动作物生长

季节倍率固定为：

- 春 `1.0`
- 夏 `1.2`
- 秋 `0.9`
- 冬 `0.0`

要求：

- 地块生长接口要升级为支持增长单位/倍率
- `TimeManager` 在每日卯时根据当前季节倍率推进地块

### 3. 季节限制进入播种逻辑

使用：

```gdscript
CropConfig.suitable_seasons
```

规则：

- 当前季节不在作物适种列表中时，不允许播种
- 提示文本必须明确，例如“当前季节不适合播种小麦”

### 4. 轻量季节视觉与 UI

本阶段只做：

- 时间显示面板继续显示季节、节气、日期、时辰、刻
- 季节切换提示弹窗
- 季节色调渐变

本阶段不做：

- 粒子系统
- 天气系统
- 复杂光照切换

---

## 必须修改的系统

### TimeManager

- 季节、节气、日期显示全部来自配置
- 存档导出新增 `year_count`
- 读旧档时兼容缺失 `year_count`
- 冬季结束进入春季时 `year_count += 1`

### Plot

- 生长接口升级为更通用增长接口
- `growth_progress` 成为主驱动值
- 每次卯时按季节倍率推进
- 冬季室外作物不自然生长

### SaveManager

- 外部 API 不变
- 能兼容读取 0.2.0 旧档
- 新存档版本升级到 0.3.0

### UI

- `TimeDisplay` 显示格式继续清晰稳定
- `SolarTermPopup` 扩展为可显示节气变化与季节切换两类提示
- `UIRoot` 或等价位置监听 `season_changed` 做色调过渡

---

## 季节文案要求

### 时间显示面板

第一行：

```text
{季节名}·{节气名} 第 {N} 天
```

第二行：

```text
{时辰名}{刻名}
```

### 季节切换弹窗

示例：

```text
夏季已到来
立夏 · 第 1 天
作物生长速度 +20%
```

---

## 四季逻辑要求

- 春季：基础倍率 `1.0`
- 夏季：倍率 `1.2`
- 秋季：倍率 `0.9`
- 冬季：倍率 `0.0`

冬季要求：

- 室外已浇水作物停止自然生长
- 不适种作物不能播种

---

## 存档兼容要求

- `TimeManager.export_save_data()` 增加 `"year_count": year_count`
- 读取 0.2.0 存档时，如果没有 `year_count`，默认使用 `0`
- 旧档中的 `season / solar_term / day_in_term / shi_chen / ke / real_play_seconds` 保持兼容

---

## 范围边界

### 本阶段必须做

- 春夏秋冬完整闭环
- 年份累计
- 四季倍率生效
- 冬季停滞与播种限制
- 时间 UI 与季节弹窗
- 色调渐变

### 本阶段不做

- 粒子系统
- 新作物完整玩法链
- 天气系统
- 事件系统

---

## 验证标准

- 春夏秋冬完整循环，冬后回春
- 夏季快于春季，秋季慢于春季，冬季停滞
- 季节切换弹窗显示正确
- 时间显示格式稳定
- 0.2.0 旧档可读取，0.3.0 新档可写入
- 睡觉跳天、暂停菜单、存读档不会因四季系统退化
