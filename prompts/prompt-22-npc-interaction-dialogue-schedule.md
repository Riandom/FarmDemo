# Phase 7-D：NPC 交互框架、基础对话与简单日程实现提示词

## 任务概述

在镇子和小屋空间稳定后，本阶段正式接入 NPC 基础层。

本阶段目标不是做复杂剧情系统，而是先让 NPC 具备最小但稳定的存在感：

- 能被识别
- 能交互
- 有基础对话
- 会按时间段切换站位或状态

这是后续好感度、送礼、委托和镇子商店的前置骨架。

---

## 核心定位

### 1. 先做统一交互框架，不做每个 NPC 各写一套

当前项目已经有交互提示和按键体系。  
NPC 必须接入同一套交互思路，而不是单独发明新入口。

玩家体验应统一为：

- 靠近 NPC
- 显示提示
- 按 `F` 交互
- 进入对话或服务界面

### 2. 第一版对话系统只求稳定，不求复杂

本阶段只做：

- 短对话
- 基础文本变化
- 同一天首次对话与重复对话差异

不做：

- 分支树
- 选项选择
- 好感度判定文本分叉
- 剧情锁和事件链

### 3. 日程只做“按时间段切换站位”

第一版 NPC 不做复杂寻路巡逻。  
只需要：

- 在不同时段出现在不同站位
- 或在同一位置切换“在岗 / 不在岗”状态

这样已经足够建立生活感，并且实现成本稳定。

---

## 交付范围

### 必做内容

- 统一 NPC 基类或统一交互协议
- 至少 3 个基础 NPC
- NPC 交互提示
- 基础对话 UI
- 按 NPC 和时间段切换基础对话
- 简单按时段站位或状态切换

### 建议第一批 NPC

- `杂货商`
- `工匠 / 铁匠类 NPC`
- `普通居民`

这样能同时覆盖：

- 商店
- 功能服务
- 日常对话

### 本阶段不做

- 好感度
- 送礼
- 委托
- 分支剧情
- 复杂寻路
- 群体 AI

---

```text

scenes/ui/

```

说明：

- 不强制一定拆成这 3 个 runtime 文件，但必须把“NPC 本体、对话、日程”职责分开
- 不应把所有 NPC 行为直接堆在每个 NPC 脚本里重复实现

---

## NPC 交互框架要求

### 统一交互行为

每个 NPC 至少要支持：

- 显示交互提示
- 被玩家锁定为当前交互目标
- 按 `F` 打开对话或服务入口

### 推荐接口

至少建议统一为：

```gdscript
func get_interaction_text() -> String
func interact(player: Node) -> void
func get_npc_id() -> String
```

### 与现有系统整合

NPC 必须兼容现有：

- 交互提示 UI
- 玩家输入桥接
- 模态 UI 管理

不要为 NPC 自己再做一套独立输入处理逻辑。

---

## 基础对话系统要求

### 第一版对话结构

每个 NPC 至少应支持：

- 默认对话
- 同一天首次对话文本
- 同一天重复对话文本
- 可按季节或时间段切换一层文本

可以采用轻量字典或资源配置，不强制现在就做复杂数据表。

### Dialogue UI 要求

第一版 UI 只需要：

- NPC 名称
- 当前文本
- 关闭或继续按钮

不需要：

- 选项分支
- 立绘
- 打字机效果
- 历史回看

### 行为要求

- 打开对话时锁定玩家常规操作
- 关闭对话后恢复控制
- 不得和背包/商店/订单 UI 同时混乱叠加

---

## 简单日程系统要求

### 日程目标

NPC 应表现出“这个世界有时间流动”，但不做重型系统。

第一版只需支持：

- 早 / 午 / 晚等时间段
- 每个时间段有一个站位点或状态

### 推荐实现方式

每个 NPC 拥有一份轻量日程定义，例如：

- `morning` -> `town_square_shop`
- `afternoon` -> `town_square_center`
- `evening` -> `home_idle`

本阶段不要求自动寻路移动。  
允许直接在时间段切换时跳到目标站位或切换可见状态。

### 与时间系统整合

日程判断应基于现有 `TimeManager`，不要再造一套时间概念。

---

## 与后续阶段的关系

本阶段完成后，`Prompt 23` 可以直接在此基础上继续：

- 好感度
- 送礼
- NPC 委托
- 镇子商店承载

所以本阶段必须保证：

- NPC 有稳定 `npc_id`
- 对话入口统一
- 日程入口统一

不要让后续系统不得不重写 NPC 基础层。

---

## 验收标准

- 至少 3 个 NPC 能稳定交互
- 靠近 NPC 有明确提示
- 按 `F` 能正常打开对话或服务入口
- 同一天首次与重复对话能区分
- NPC 会按时间段切换站位或状态
- 对话 UI 与现有模态体系兼容
- 不破坏农场、洞窟、背包、商店和订单系统

---

## 实现建议

### 建议优先顺序

1. 先做 NPC 通用交互基类
2. 再做对话 UI 和最小文本结构
3. 再接 3 个基础 NPC
4. 最后接简单按时段站位

### 推荐做法

- 先把交互协议统一
- 对话先用轻量结构跑通
- 日程先做站位切换，不做复杂移动

### 不推荐做法

- 每个 NPC 各写一套交互逻辑
- 一开始就做分支剧情编辑器
- 一开始就做复杂寻路巡逻
