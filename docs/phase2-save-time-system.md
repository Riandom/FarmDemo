# 第二阶段开发需求文档：存档系统与春季时间系统

## 📋 文档概述

**目标**：为 FarmDemo 添加完整的存档系统和基于中国传统历法的时间系统，使游戏从 Demo0.1 升级到 Demo0.2。

**核心功能**：
- ✅ JSON 格式存档/读档系统
- ✅ 二十四节气时间系统（春季 6 节气 × 7 天 × 12 时辰 × 4 刻）
- ✅ 床家具（快速跳过当天 + 自动存档）
- ✅ UI 显示当前日期、时辰、节气

---

## 🎯 一、存档系统设计

### 1.1 功能描述

实现一个面向未来的存档系统，能够完整保存和恢复游戏状态，支持后续扩展更多游戏内容。

**设计原则**：
- **完整性**：保存所有影响游戏进度的数据
- **可扩展性**：新增功能时无需大幅修改存档结构
- **安全性**：防止存档损坏导致数据丢失
- **调试友好**：使用 JSON 格式便于人工查看和调试

### 1.2 数据结构定义

#### 存档文件结构（JSON 格式）

```json
{
  "version": "0.2.0",                    // 游戏版本号，用于兼容判断
  "save_timestamp": "2026-04-03T15:30:00Z", // 现实世界存档时间戳
  "game_state": {
    "player": {
      "position_x": 128.5,               // 玩家 X 坐标
      "position_y": 256.0,               // 玩家 Y 坐标
      "gold": 150,                       // 金币数量
      "inventory": {                     // 背包物品
        "seed_wheat": 10,
        "crop_wheat": 5,
        "hoe_wood": 1
      },
      "unlocked_tools": ["hoe_wood", "watering_can_basic"],
      "current_tool": "hoe_wood"
    },
    "time": {
      "season": "spring",                // spring/summer/autumn/winter
      "solar_term": 2,                   // 当前节气索引 (0-5)
      "day_in_term": 3,                  // 节气内第几天 (0-6)
      "shi_chen": 5,                     // 当前时辰 (0-11, 0=子时)
      "ke": 2                            // 当前刻 (0-3)
    },
    "farm": {
      "plots": [                         // 36 个地块状态数组
        {
          "grid_x": 0,
          "grid_y": 0,
          "base_state": "plowed",        // waste/plowed/seeded/watered/mature
          "growth_stage": 2,             // 0-3 生长阶段
          "growth_progress": 0.65,       // 0.0-1.0 阶段进度
          "crop_config_id": "crop_wheat"
        },
        // ... 更多地块
      ]
    }
  }
}
```

#### 关键字段说明表

| 字段路径 | 数据类型 | 说明 | 示例值 |
|---------|---------|------|-------|
| `version` | String | 游戏版本号，用于存档兼容性判断 | `"0.2.0"` |
| `game_state.player.position_x/y` | Float | 玩家在世界中的坐标 | `128.5` |
| `game_state.player.gold` | Integer | 玩家当前金币 | `150` |
| `game_state.player.inventory` | Dictionary | 背包物品字典 `{物品 ID: 数量}` | `{"seed_wheat": 10}` |
| `game_state.time.season` | String | 当前季节 | `"spring"` |
| `game_state.time.solar_term` | Integer | 节气索引（该季节内） | `2` = 惊蛰 |
| `game_state.time.day_in_term` | Integer | 节气内的第几天 | `3` = 第 4 天 |
| `game_state.time.shi_chen` | Integer | 时辰索引 | `5` = 辰时 (7:00-9:00) |
| `game_state.time.ke` | Integer | 刻索引 | `2` = 第 3 刻 |
| `game_state.farm.plots[]` | Array | 所有地块状态数组 | 见左侧 |

### 1.3 存档触发机制

#### 必须实现的存档时机

| 触发条件 | 优先级 | 说明 |
|---------|-------|------|
| **与床互动** | 🔴 必需 | 玩家按 E 键与床互动后立即存档 |
| 退出游戏前 | 🟡 推荐 | 检测场景树的 `tree_exiting` 信号 |
| 手动存档 | 🟢 可选 | 预留快捷键（如 F5）供测试使用 |

#### 存档流程图（文字描述）

```
触发存档事件
    ↓
暂停游戏逻辑（防止存档过程中状态变化）
    ↓
收集当前游戏状态
    ├── GameManager: 金币、背包、工具
    ├── TimeManager: 日期、时辰、刻
    ├── FarmManager: 遍历所有地块状态
    └── Player: 当前位置
    ↓
序列化为 JSON 对象
    ↓
写入临时文件 save_temp.json
    ↓
验证临时文件完整性
    ↓
删除旧存档 save.json（如果存在）
    ↓
重命名临时文件为 save.json
    ↓
恢复游戏逻辑
    ↓
显示"已存档"提示（0.5 秒）
```

### 1.4 读档流程

```
请求读档
    ↓
检查 save.json 是否存在
    ↓
解析 JSON 文件
    ↓
校验 version 字段兼容性
    ↓
恢复 GameManager 状态（金币、背包）
    ↓
恢复 TimeManager 状态（日期、时辰）
    ↓
恢复 FarmManager 状态（重建地块）
    ↓
恢复 Player 位置
    ↓
发送"读档完成"信号
    ↓
刷新所有 UI 显示
```

### 1.5 错误处理要求

| 错误场景 | 处理方式 |
|---------|---------|
| 存档文件不存在 | 不报错，视为新游戏 |
| JSON 格式损坏 | 弹出提示"存档损坏，无法读取"，开始新游戏 |
| version 不兼容 | 提示"存档版本过旧，无法读取" |
| 缺少必填字段 | 使用默认值填充，记录警告日志 |
| 写入失败（磁盘满） | 弹出提示"存档失败，请检查存储空间" |

## ⏰ 二、春季时间系统

### 2.1 时间层级结构（比喻：俄罗斯套娃）

**最大层 → 最小层**：
```
年 → 季节 → 节气 → 天 → 时辰 → 刻
 春     立春    D1    子时   初刻
 夏     雨水    D2    丑时   二刻
 秋     惊蛰    D3    寅时   三刻
 冬     春分    D4    卯时   四刻
       清明    D5
       谷雨    D6
```

#### 时间单位换算表

| 游戏单位 | 包含关系 | 现实时长 | 说明 |
|---------|---------|---------|------|
| **1 刻** | 基础单位 | 30 秒 | 最小时间流逝单位 |
| **1 时辰** | 4 刻 | 2 分钟 | 古代 2 小时制（子丑寅卯...） |
| **1 天** | 12 时辰 | 24 分钟 | 完整的一天循环 |
| **1 节气** | 7 天 | 2.8 小时 | 每个节气持续一周 |
| **1 季节** | 6 节气 | 16.8 小时 | 春季 = 立春→谷雨 |
| **1 年** | 4 季节 | 67.2 小时 | 完整四季轮回 |

### 2.2 二十四节气名称映射

#### 春季专属节气（Phase 2 仅实现春季）

| 索引 | 节气名称 | 含义说明 | 对应公历 |
|-----|---------|---------|---------|
| 0 | **立春** | 春季开始，万物复苏 | 2 月 3-5 日 |
| 1 | **雨水** | 降雨增多，适合播种 | 2 月 18-20 日 |
| 2 | **惊蛰** | 春雷惊醒冬眠动物 | 3 月 5-7 日 |
| 3 | **春分** | 昼夜平分，春天中点 | 3 月 20-22 日 |
| 4 | **清明** | 气候清爽，扫墓祭祖 | 4 月 4-6 日 |
| 5 | **谷雨** | 雨水滋润，谷物生长 | 4 月 19-21 日 |

> 📌 **注意**：Phase 2 只需实现春季 6 节气，但代码结构要预留夏、秋、冬三季的扩展接口。

### 2.3 十二时辰与现代时间对照

| 时辰索引 | 时辰名 | 现代时间范围 | 游戏内显示 | 说明 |
|---------|-------|------------|-----------|------|
| 0 | 子时 | 23:00-01:00 | 子时 | 深夜 |
| 1 | 丑时 | 01:00-03:00 | 丑时 | 凌晨 |
| 2 | 寅时 | 03:00-05:00 | 寅时 | 黎明前 |
| 3 | **卯时** | **05:00-07:00** | **卯时** | **天亮，一天开始** |
| 4 | 辰时 | 07:00-09:00 | 辰时 | 早晨 |
| 5 | 巳时 | 09:00-11:00 | 巳时 | 上午 |
| 6 | 午时 | 11:00-13:00 | 午时 | 正午 |
| 7 | 未时 | 13:00-15:00 | 未时 | 下午 |
| 8 | 申时 | 15:00-17:00 | 申时 | 傍晚前 |
| 9 | **酉时** | **17:00-19:00** | **酉时** | **日落，天黑** |
| 10 | 戌时 | 19:00-21:00 | 戌时 | 夜晚 |
| 11 | 亥时 | 21:00-23:00 | 亥时 | 深夜 |

> 📌 **关键时间点**：
> - **卯时（5:00）**：天亮，作物开始生长
> - **酉时（17:00）**：天黑，视觉上需要变暗
> - 玩家可见时间显示：**只显示时辰**（如"辰时"），不显示数字时间（如"08:30"）

### 2.4 时间推进规则

#### 时间流逝公式

```
现实 30 秒 = 游戏 1 刻
游戏 1 天 = 12 时辰 × 4 刻 × 30 秒 = 现实 24 分钟
```

#### 时间推进伪逻辑（用自然语言描述）

```
每过 30 秒现实时间：
    刻数 + 1
    如果 刻数 == 4:
        刻数 = 0
        时辰 + 1
        如果 时辰 == 12:
            时辰 = 0  # 回到子时
            天数 + 1
            如果 天数 == 7:  # 一个节气结束
                天数 = 0
                节气索引 + 1
                如果 节气索引 == 6:  # 春季结束
                    节气索引 = 0
                    季节切换为下一个（Phase 2 暂不实现）
```

### 2.5 作物生长与时间的关系

#### 生长速率计算

当前 Phase 1 使用固定 Timer（5 秒/阶段），Phase 2 改为基于游戏时间：

| 作物类型 | 生长所需天数 | 相当于现实时间 | 说明 |
|---------|------------|--------------|------|
| 小麦（基础） | 3 天 | 72 分钟 | 从播种到成熟 |
| 快速作物 | 1-2 天 | 24-48 分钟 | 测试用或低级作物 |
| 慢速作物 | 5-7 天 | 120-168 分钟 | 高级作物 |

#### 生长触发时机

```
每个卯时（5:00 AM）到来时：
    遍历所有处于"已浇水"状态的地块
    增加生长进度
    如果 生长进度 >= 1.0:
        生长阶段 + 1
        重置生长进度
        如果 生长阶段 >= 最大阶段:
            地块状态变为"成熟"
```

## 🛏️ 三、床家具系统

### 3.1 功能描述

床是一个可交互的家具，玩家与之互动可以：
1. **快速跳过当天剩余时间**
2. **直接进入第二天的卯时（5:00 AM）**
3. **自动保存游戏进度**

### 3.2 床的场景结构

```
Bed (Area2D)
├── Sprite2D (床的贴图)
├── CollisionShape2D (碰撞体，检测玩家进入)
└── InteractionPrompt (子节点，显示"按 E 休息")
```

### 3.3 交互流程

```
玩家走到床边（进入 Area2D 范围）
    ↓
显示交互提示："按 E 休息"
    ↓
玩家按下 E 键
    ↓
调用 SaveManager.save_game()
    ↓
TimeManager.set_next_day_mao_hour()
    ↓
屏幕淡出效果（可选，黑色矩形透明度 0→255→0）
    ↓
等待 0.3 秒
    ↓
屏幕淡入
    ↓
显示提示："新的一天开始了！卯时"
    ↓
刷新 UI 时间显示
```

### 3.4 时间跳跃规则

| 输入条件 | 输出结果 |
|---------|---------|
| 任何时辰与床互动 | 第二天卯时（时辰=3, 刻=0） |
| 日期变化 | 天数 + 1（节气内天数推进） |
| 节气变化检测 | 如果天数>=7，进入下一节气 |
| 作物生长触发 | 在卯时统一计算生长进度 |

### 3.5 床的放置位置

- **Phase 2 默认**：在玩家小屋内预置一张床（固定位置）
- **未来扩展**：允许玩家移动/购买不同样式的床

## 🖥️ 四、UI 显示规范

### 4.1 新增 UI 组件

需要在现有 UI 系统中添加以下组件：

#### 时间显示面板（TimeDisplay）

**位置**：屏幕右上角（不遮挡农场和玩家）

**显示内容**：
```
┌─────────────────┐
│ 春季·谷雨 第 4 天 │
│     辰时三刻     │
└─────────────────┘
```

**字段说明**：
- 第一行：`{季节名}·{节气名} 第{N}天`
- 第二行：`{时辰名}{刻名}`（刻名：初刻/二刻/三刻/四刻）

**刷新频率**：每当刻数变化时刷新（每 30 秒现实时间）

#### 节气提示弹窗（SolarTermPopup）

**触发条件**：进入新节气时（day_in_term == 0）

**显示效果**：
```
┌──────────────────────┐
│                      │
│    今日谷雨          │
│  2026 年 4 月 19 日     │
│                      │
└──────────────────────┘
```

**持续时间**：3 秒后自动消失

**动画**：从屏幕顶部滑入 → 停留 3 秒 → 向上滑出

### 4.2 UI 更新协议

```
TimeManager 发出 signal_time_changed() 信号
    ↓
TimeDisplay 监听信号
    ↓
调用 update_display(season, solar_term, day, shi_chen, ke)
    ↓
格式化文本并更新 Label
```

## 🧩 五、扩展性设计说明

### 5.1 夏季、秋季、冬季的扩展方式

Phase 2 只实现春季，但代码必须预留扩展接口：

#### 季节配置表（未来扩展）

| 季节名 | 节气列表（6 个） | 代表作物 | 特殊机制 |
|-------|----------------|---------|---------|
| **春** | 立春、雨水、惊蛰、春分、清明、谷雨 | 小麦、草莓 | 基础生长速度 |
| **夏** | 立夏、小满、芒种、夏至、小暑、大暑 | 玉米、西瓜 | 生长速度 +20% |
| **秋** | 立秋、处暑、白露、秋分、寒露、霜降 | 水稻、南瓜 | 收获产量 +10% |
| **冬** | 立冬、小雪、大雪、冬至、小寒、大寒 | ❄️休耕 | 无法种植室外作物 |

#### 扩展方法

当未来添加夏季时，只需：
1. 在 `seasons` 数组添加 `"summer"`
2. 在 `solar_terms` 字典添加夏季 6 节气名称
3. 调整时间推进逻辑支持跨季节循环
4. 添加夏季专属作物配置

### 5.2 存档版本兼容性

使用 `version` 字段管理存档格式演进：

```
v0.2.0: 初始版本（春季 + 存档）
v0.3.0: 添加夏季（向后兼容：旧存档夏季默认为春季）
v0.4.0: 添加 NPC 好感度（向后兼容：旧存档好感度初始化为 0）
```

#### 版本升级策略

```
如果 存档.version < 当前游戏.version:
    执行版本升级迁移
    添加缺失字段（使用默认值）
    保存为新格式
否则：
    正常读取
```

### 8.2 时间配置参数（time_config.tres）

```
刻的现实时长：30.0 秒
每天开始时辰：3 (卯时)
每天结束时辰：9 (酉时)
春季月份范围：[2, 3, 4]  # 用于节气弹窗显示
```

### 8.3 信号定义建议

**TimeManager 信号**：
- `signal time_changed(shi_chen: int, ke: int)`
- `signal day_changed(day_in_term: int)`
- `signal solar_term_changed(solar_term_index: int)`

**SaveManager 信号**：
- `signal save_completed(success: bool)`
- `signal load_completed(success: bool)`
