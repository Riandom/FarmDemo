# Phase 6: 飞镖出征式战斗与洞窟探索原型

## 文档概述

**目标**：在 Phase 5 农场经营深化完成后，为 FarmDemo 建立第一条真正成立的战斗探索闭环。  
**版本定位**：Demo `0.7.x`  
**前置条件**：Phase 5 的体力、多作物、订单、背包/热栏/商店 UI 已完成并通过基础验证。  
**Phase 范围**：仅包含 3 个方向

- Phase 6-A：俯视即时战斗最小闭环
- Phase 6-B：小型洞窟探索与撤退结算
- Phase 6-C：经济主绑定的战斗准备系统

本阶段不是“全面战斗系统扩张”，也不是完整镇子/NPC 社交阶段。  
本阶段的正确目标是：先把“卖货赚钱 -> 采购飞镖 -> 下洞战斗 -> 带回掉落”的闭环做出来。

---

## 核心定位

### 1. 不照抄星露谷，但仍保持俯视即时战斗

当前项目已经稳定在：

- `CharacterBody2D`
- 2D 俯视四方向移动
- 主场景直接操作
- 热栏/背包/商店/时间/存档一体化

因此本阶段不应为了“做出差异”强行改成：

- 横板平台战斗
- 回合制战斗场景切换
- 独立局内肉鸽模式

真正的差异不放在视角，而放在：

- 战斗准备结构
- 资源消耗方式
- 洞窟推进节奏
- 农场和战斗的经济绑定

### 2. 战斗特色应是“出征式消耗型武器”

本项目的第一版战斗武器不应是“长期拿着一把稳定武器反复挥”。  
特色必须明确落在：

- `高消耗`
- `低耐久`
- `投射型优先`

第一把核心原型武器就是：

- `飞镖`

玩家进入洞窟前，需要先完成一轮准备：

1. 卖出农场产物获得金币
2. 在战斗 NPC 处购买飞镖与基础补给
3. 决定本次要带多少飞镖、是否接受损耗
4. 进入洞窟战斗与采集

这条准备过程本身，就是本项目和常规种田战斗游戏拉开差异的关键。

### 3. 采用“经济主绑定 + 少量直接材料”而非纯数值买剑

战斗与农场的关系不能只是：

> 卖菜换钱买更强武器

这样会太普通。  
正确做法应是：

- 大部分战斗资源来自经济采购
- 少量关键强化来自农场产物与洞窟材料共同驱动

例如：

- 普通飞镖可以直接购买
- 强化飞镖、毒镖、爆裂镖需要额外材料
- 某些战斗配件需要洞窟掉落 + 农场产物共同解锁

这样战斗既依赖经济循环，又保留独特身份。

---

## Phase 6-A：俯视即时战斗最小闭环

### 目标

在不破坏现有俯视种田底座的前提下，建立一套最小可运行的实时战斗原型。

### 核心交付

- 玩家可在探索场景中移动和攻击
- 第一把投射武器 `飞镖`
- 飞镖具备数量消耗
- 武器本体具备低耐久或明显补给压力
- 至少 2 种敌人
- 玩家受伤与撤退逻辑

### 本阶段不做

- 复杂连段
- 闪避翻滚系统
- 近战/远程/法术全并行
- 多角色职业
- 华丽技能树

### 工程原则

- 优先复用现有玩家移动和输入结构
- 战斗输入必须兼容当前热栏体系
- 飞镖不做成临时测试武器，而是 Phase 6 的核心身份原型

---

## Phase 6-B：小型洞窟探索与撤退结算

### 目标

让战斗不只是“房间里打怪”，而是形成一条可进入、可推进、可撤退、可结算的探索链路。

### 核心交付

- 1 张小型洞窟地图
- 明确入口和返回逻辑
- 洞窟中存在敌人和基础资源点
- 玩家可主动撤退或战败后被送回安全区
- 返回后保留合理收益，并承担轻惩罚

### 推荐失败模型

- 主动撤退：安全返回，保留大部分收益
- 战败返回：损失少量金币、飞镖或当次收益
- 不采用高惩罚死亡模式

### 本阶段不做

- 多层矿洞
- 随机地图生成
- 高压永久损失
- 复杂路径谜题

### 工程原则

- 第一版先把“进洞 -> 战斗 -> 回家”跑通
- 洞窟应是农场主循环的延伸，不是独立小游戏

---

## Phase 6-C：经济主绑定的战斗准备系统

### 目标

让战斗强度明显取决于农场经营准备，而不是只靠角色裸模数值。

### 核心交付

- 至少 1 个战斗相关 NPC
- 该 NPC 负责出售飞镖、补给、基础维修或武器升级
- 玩家通过卖作物获得进入战斗循环的经济能力
- 洞窟掉落可用于后续强化飞镖或解锁下一层战斗内容

### 正确的资源流

应形成以下结构：

1. 农场产出 -> 售卖 -> 金币
2. 金币 -> 战斗 NPC -> 飞镖与补给
3. 洞窟探索 -> 掉落材料
4. 材料 + 金币 -> 更稳定的出征能力

### 本阶段不做

- 完整镇子生态
- 多个商店职业分工
- 完整制造系统
- 大量战斗支线委托

### 工程原则

- 先有 1 个战斗供应 NPC 即可
- 不要一开始做复杂镇子层
- 采购、补修、补充飞镖要足够明确，能让玩家感知“这次出征是一次经营决策”

## 与当前系统的正确关系

### 1. GameManager

Phase 6 新增的战斗资源、金币流转、飞镖数量和掉落进入背包时，仍应优先围绕 `GameManager` 现有物品接口扩展。  
不要重新做一套战斗背包。

### 2. Hotbar / Inventory / Shop

现有 Prompt 17 的背包和热栏系统，是 Phase 6 的直接前置。  
飞镖和战斗补给应当：

- 可以进入背包
- 可以进入热栏
- 可通过现有 UI 管理

但战斗 NPC 商店不应直接复用“农场种子商店”逻辑到完全混杂。  
允许在 Phase 6 内新增战斗商店界面或在现有商店结构上分离战斗供应页。

### 3. TimeManager / SaveManager

洞窟探索必须继续受当前时间和读档体系管理。  
战斗不是旁路模式：

- 下洞应消耗当天时间
- 返回后仍回到主时间轴
- 飞镖消耗、掉落收益、耐久状态必须进入存档

### 4. Main / Scene Flow

本阶段应开始出现“农场主场景之外的第二类主玩法场景”。  
但切换方式必须保持简单可控：

- 从主场景进入洞窟
- 从洞窟返回主场景

不要一开始就铺多层复杂场景管理器。

---

## 验证目标

### Phase 6-A 完成后

- 玩家能在俯视视角中进行基础实时战斗
- 飞镖能消耗、命中、失去或回收
- 至少两种敌人行为可区分

### Phase 6-B 完成后

- 洞窟能进入和返回
- 战败与撤退结算成立
- 玩家明确感受到“本次出征资源是有限的”

### Phase 6-C 完成后

- 农场收入能支撑战斗准备
- 战斗准备明显影响下洞表现
- 战斗收益能反哺后续经营和战斗成长

---

## 结论

Phase 6 的正确路线不是：

- 复制星露谷的近战挥剑
- 或为求差异强行换成别的游戏类型

而是：

- 保留俯视即时战斗底座
- 以飞镖为第一把核心武器
- 让玩家通过种田经济完成每次出征准备
- 用“高消耗、低耐久、投射型、轻惩罚撤退”的结构做出身份差异

这样才能既和当前仓库兼容，又真正开始形成这款游戏自己的战斗气质。
