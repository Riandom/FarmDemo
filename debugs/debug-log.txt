静态发现的问题：
1. 5 个 autoload 相关脚本仍保留 class_name，存在 singleton hides 风险
2. 多个脚本仍直接调用 FarmManager./GameManager./FarmInteractionSystem.，存在 non-static function 调用风险
3. texture_generator.gd 依赖 ColorPalette 类型名；删除 class_name 后会继续报类型错误

本轮修改：
1. 删除以下脚本中的 class_name：
   - scripts/systems/farm_manager.gd
   - scripts/systems/farm_interaction_system.gd
   - scripts/systems/farm_render_system.gd
   - scripts/systems/game_manager.gd
   - scripts/resources/color_palette.gd
2. 将以下脚本中的单例调用改为 /root/... 实例访问：
   - scripts/main.gd
   - scripts/player/player_input_bridge.gd
   - scripts/ui/interaction_prompt.gd
   - scripts/plot/plot.gd
   - scripts/systems/farm_render_system.gd
3. 将单例缓存变量改为动态引用，避免 Node 静态类型导致的成员解析错误
4. 调整 scripts/tools/texture_generator.gd，移除对 ColorPalette 类型名的强依赖
5. 调整 resources/config/placeholder_colors.tres，移除 script_class=ColorPalette

静态验证结果：
1. 搜索 `class_name FarmManager|FarmInteractionSystem|FarmRenderSystem|GameManager|ColorPalette`：无结果
2. 搜索 `FarmManager.` / `GameManager.` / `FarmInteractionSystem.` / `FarmRenderSystem.`：无结果
3. 占位 PNG 数量：32
